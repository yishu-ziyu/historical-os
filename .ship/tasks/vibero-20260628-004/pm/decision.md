# Round 046 PM 决策 — 产品定位（AI 接管版）

## 决策：已定 — 不再重审，直接进入「档案值班台」情感化补强

用户授权 AI 全权判断，明早验收。本决策由 AI 基于 43 轮共创沉淀 + 现有 demo 实读后做出。

## 一句话定位

> **你是一个时间档案值班台情报官。历史裂缝每天落到你桌上。读档案、标线索、做小决策——世界线随你可见地漂移。你不是在解谜，你是在守门。**

## 三个视角对比

| 视角 | 判断 |
|------|------|
| 用户原话（round 044 阻塞） | "取决于游戏到底想怎么玩" → 想要"好玩、好看"，不要"机制正确但情感平" |
| ChatGPT 视角（未实际取回） | 代码事实=档案 OS + 三段回合 + 标记线索，但没回答"应不应该这样" |
| AI 接管判断 | 机制已经对了，缺的是**情感钩子**和**可见后果**——不是重做，是补强 |

## 现有 demo 的真实问题（诊断）

读完 `index.html` / `turn_cycle.js` / `server.mjs` 后，问题不是"游戏想怎么玩"不清楚，而是：

1. **第一屏无情感钩子** — 用户打开看到「一条异常已经进入值班队列」+ 一个按钮。这是机制描述，不是情感入口。15 秒内没有"我想知道接下来会发生什么"的冲动。
2. **选择后果不可见** — 玩家选了一个 action，aftermath 给出叙事 + stateChanges，但**世界线漂移不可见**。玩家无法感知"我的选择撬动了什么"。
3. **没有视觉化的累积感** — case state 累积了 nodes/edges/risk，但前端没有持续可见的"世界线状态"指示器。每回合像是独立的，不像是在塑造一条世界线。
4. **没有 arc 收束** — 可以无限继续，但没有"这一案告一段落"的感觉。5-7 回合后应该有 case closure 的视觉化总结。

## 决策

**不重做。** 43 轮沉淀的机制（三段回合 + LLM + case state + 标记线索 + 历史框架）是对的。

**补强三件事**（round 047 实现）：

1. **开场序列（opening cinematic）** — 替换当前 empty state。黑屏 → 打字机 → 日期 → 电报 → 世界线漂移指示器动画 → desk 渐入。15 秒内建立情感。
2. **可见的世界线漂移（worldline drift indicator）** — 顶部常驻一条横向漂移条。每回合根据 stateChanges 漂移。颜色：绿（接近真实）→ 琥珀 → 红（重大分叉）。让选择有视觉重量。
3. **回合后果可视化（stakes panel）** — aftermath 段显示「本回合世界线偏移 +Xσ」「累计漂移 +Yσ」「下一回合风险：Z」。让玩家看到自己撬动了什么。

## 范围

**Out**:

## 优先级：P0

比 round 044（Playwright 视觉验证）优先级更高。是上游决策。

## 风险

- 开场序列可能让老用户觉得"又多了一步"。**对策**：开场只在首次访问播放，sessionStorage 标记后不再播。
- 漂移指标的计算口径需要明确。**对策**：基于 case state 的 `stateChangeHistory.length` + `riskOverall`，公式简单可解释。
- 后果可视化可能让"模糊后果"的美感消失。**对策**：只显示量级，不显示方向（玩家不知道是好是坏，只知道"撬动了"）。

## 验收标准（明早用户）

1. 打开 `http://127.0.0.1:8892/`，15 秒内有情感反应（不是"哦一个 UI"，而是"咦发生了什么"）
2. 完成一回合后，能说出"我的选择让世界线漂移了 X"
3. 三回合后，能感知到"我在塑造一条世界线"
4. 现有功能（标记线索、档案柜、三段回合）全部仍工作
