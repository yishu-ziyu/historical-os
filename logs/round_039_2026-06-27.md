# Round 039 - 真实模型接入测试

> 日期：2026-06-27
> 主题：端到端验证三段式回合循环 + 真实 LLM 模型

## 发现的问题

`callAnthropicMessages` 之前在内部调用了 `parseModelText(text)`，会把 turn cycle JSON 提前解析成 legacy 格式（需要 `story` / `choices` 字段）。三段式 prompt 输出的是 `intelCard` / `situationRoom` / `stateChanges`，匹配不到导致 fallback。

## 修复

1. **`callAnthropicMessages` / `callOpenAICompatibleChat`** 现在返回 **raw text**，由调用方决定用哪个 parser。
2. **`callModel`（legacy）** 显式包了 `parseModelText(text)` 维持向后兼容。
3. **`callModelWithPrompt`**（三段式）使用 schema-aware parsers（`parseBriefingResult` / `parseSituationRoomResult` / `parseAftermathResult`）。
4. **Schema 验证加严**：`assertFields()` 在缺关键字段时抛错，触发 fallback 而非 silent 降级到占位数据。
5. **生成参数调优**：三段式 prompt 用 `max_tokens=2000` + `temperature=0.5`（比 legacy 的 1200/0.9 更稳定）。

## 测试结果

- 4 个 turn_cycle 后端测试：全绿
- 9 个 turn_cycle_frontend 测试：全绿
- 5 个 legacy server.test.mjs：3 通过，2 预存失败（端口竞争，与本次改动无关）
- 1 个 timeout 测试：偶发端口竞争
- **真实模型 e2e：成功**（情报卡 + 态势面板 + Aftermath 三个阶段都生成了符合 schema 的真实数据）

## 关键收获

1. **历史事实锚定有效**：模型能根据历史事实生成有可信度的档案体文本，包括日期、编号、人名、机构、法律法规。
2. **schema 严格性有效**：tight schema + temperature 0.5 + max_tokens 2000 让模型稳定按结构输出，不再跑偏到自由续写。
3. **状态变更可执行**：模型输出的 `stateChanges` 用 `target.field: from → to` 格式，可直接被引擎执行更新世界线状态。
