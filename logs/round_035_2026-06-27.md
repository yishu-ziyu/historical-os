# Round 035 - 混合式三段展开 + 竞品调研

> 日期：2026-06-27
> 主题：回合循环设计 + 四类参考游戏深度调研

---

## 调研结论

本轮针对四类参考游戏做了深度调研，提炼出对项目最可落地的设计机制。

### TNO（The New Order: Last Days of Europe）

**超事件系统（Super Event）**
- 异常事件不是普通弹窗，而是带有多层后果的历史转折点
- 有三层后果结构：即时后果 → 中间层后果 → 长期后果
- 已被各大 MOD 借鉴，说明是被验证过的模式
- 对项目的含义：异常事件 = 超事件，需要定义"后果骨架"（手工写，保证可信度），LLM 在骨架内填充血肉

**封锁而非开放**
- TNO 封锁了原版外交系统大部分选项，防止玩家"串戏"
- 项目的 HistoryGuard 层本质上就是做这件事

**多视角事件链**
- 同一历史转折点，不同身份看到的事件链完全不同
- 对应项目的产品判断：调查者/外交官/情报人员是同一个异常的不同接口

**文案量级**
- TNO 事件文本"下到平民百姓，上到国家领袖"
- 十年开发、7 年 beta——架空历史的可信度来自海量手工事件文本
- LLM 可以辅助生成，但核心事件文案锚点必须人工把关
- "边写边改"拖慢更新速度——先冻结内容标准，再批量生成

### Kaiserreich

**事件文本即线索**
- 事件文案本身就应该有推理价值
- 不直接告诉玩家"发生了什么"，而是让玩家从文本里拼出来
- 差的写法："爱因斯坦没有离开德国，这很危险"
- 好的写法：有日期、有文件名、有手写注释、有矛盾点的真实档案格式

### Papers Please

**操作即叙事**
- 盖章、标记、归档这些有物理感的操作就是叙事本身
- 不需要额外的过场动画
- 对应项目：情报卡上的「标记线索」「验证来源」「发送指令」都应有体感

**政策公告板**
- 每天更新政策公告，直接改变审核标准
- 项目的 Timeline 历史偏移应该直接改变后续情报卡的可信度判断标准

**时限压力（叙事性）**
- 不是倒计时条，是报纸出刊、电报到达、会议召开的叙事性截止线

### Disco Elysium

**文本排版即玩法**
- 对话/选项放右下角（右手活动区）
- Twitter 式可滚动长文本框
- 衬线字体 + 适当行距 + 段间距 = 阅读舒适度
- 1+ million words 脚本——文案总量是核心体验

**内部声音即 UI 元素**
- 主角的内心冲突做成可视化的 UI 元素
- 项目的"情报分析过程"也应有"内部声音"——像两个分析师在档案室里争论的对话气泡

## 核心设计决策

### 1. 一回合 = 一次决策，三段展开

不是"多步操作自然涌现"，而是参照 TNO / Papers Please / Disco Elysium 的共同模式：
**一次聚焦的决策，三段有节奏的反馈。**

### 2. 三段结构

| 阶段 | 名称 | 时长 | 核心机制 |
|------|------|------|---------|
| 第一段 | 情报卡（The Briefing） | 2-3 分钟 | 阅读、标记线索、验证来源 |
| 第二段 | 沙盘推演（The Situation Room） | 1-2 分钟 | 选择行动方向、看后果预览 |
| 第三段 | LLM 叙事推演（The Aftermath） | 3-5 分钟 | 阅读叙事、看世界线偏移、接收新情报卡 |

### 3. 情报卡文案标准 = 「档案体」

用户确认喜欢的写法：
- 有编号、日期、来源
- 正文像真实档案记录
- 有手写注释、矛盾点
- 不直接告诉玩家"发生了什么"，让玩家从文本拼出来
- 中英混杂（档案原文保留德语/英语，玩家界面中文）

### 4. 操作体感

参照 Papers Please 的"盖章感"：
- 标记线索 = 高亮/标注
- 验证来源 = 交叉比对操作
- 发送指令 = 有物理反馈的动作

### 5. LLM 输出必须附带状态声明

参照 Story2Game 论文：
- 每次叙事输出都附带可执行的状态声明
- 不是"写了一段好看的话"，而是"爱因斯坦状态变成犹豫中，风险+1"
- 引擎执行这些声明，影响后续所有情报卡

### 6. 自由文本输入受约束

参照 Hidden Door：
- 不符合历史框架的选项不被直接拒绝
- 给出条件反馈："这个方向需要额外条件"
