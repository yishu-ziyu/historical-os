# Round 019 - 第一屏布局确认：默认工作台 + 可自由拖拽窗口

日期：2026-05-18

## 本轮问题
情报工作台第一屏的布局应该如何安排？

建议布局：

```text
┌────────────────────────────────────────────────────┐
│ Historical OS | 1933-03-XX | Baseline Drift: HIGH  │
├───────────────┬────────────────────────┬───────────┤
│ Timeline      │ Intel Desk              │ Map       │
│ 异常节点       │ 主情报卡                 │ 相关地点   │
│ 倒计时压力     │ 五个动作 + 弱推荐          │ 路线/标记  │
├───────────────┴────────────────────────┴───────────┤
│ Archives / Source Credibility / Risk Alerts        │
└────────────────────────────────────────────────────┘
```

## 用户回答
用户支持这个布局基本正确，但希望它可以自由拖拽。

用户还指出：在更大一层级的 `黑客松` 文件夹里，正在做一个和游戏开发有关的项目，其中有一个游戏引擎。现在可以尝试跑简单 Demo，需要给一个具体 prompt，让另一边引擎能够对接。

## 本轮确认判断
第一屏采用：

> 默认情报工作台布局 + OS 式自由拖拽窗口。

也就是说：
- 初始打开时，系统给一个合理默认布局。
- Intel Desk 仍是中心最大焦点。
- Timeline / Map / Archives / Risk 是周边面板。
- 用户可以拖动、调整、展开、最小化窗口。
- 这强化 OS 感，同时不牺牲第一分钟的信息秩序。

## 产品含义
第一屏不能一开始就是混乱桌面；但也不能是死板仪表盘。

最佳形态是：

```text
系统默认排好一个危机工作台
       ↓
用户可以像操作 OS 一样重排窗口
       ↓
布局本身逐渐成为玩家的工作方式
```

## 下一步
