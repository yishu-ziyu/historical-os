# Round 025 - 修复 Web Demo 故事网回退与候选重复

日期：2026-05-18

## 用户反馈
用户指出 Web Story Loop Demo v0.1 存在两个问题：

1. 当前已选择的“跟进纳粹宣传机器”仍然出现在候选分支中，理论上应隐藏。
2. “故事网”只是展示节点，不能点击回退查看，因此不是真正的故事网。

## 本轮修复
修改文件：

```text
artifacts/web_story_loop_demo/script.js
artifacts/web_story_loop_demo/style.css
```

修复内容：

1. 生成下一组选项时，过滤已在历史路径中出现过的节点标题，避免当前路径或已走过路径再次作为候选出现。
2. 故事网节点从静态 `div` 改为可点击 `button`。
3. 点击故事网节点时，会跳转回该节点，并截断其后的路径，让用户可以从旧节点重新分叉。
4. 为故事网节点增加 hover 样式，暗示它可以点击。

## 产品含义
故事网不应该只是“看起来像节点”，而应该真正支持：

- 回到某个旧节点。
- 重新选择另一条路径。
- 删除该节点之后的未来。
- 让玩家感觉自己在探索网状历史，而不是阅读线性记录。

## 验证
页面仍可通过 cmux 浏览器加载，标题为：

```text
网状历史故事书 Demo
```

## 下一步
继续验收：故事节点点击回退是否符合用户预期；若成立，下一步再做“分支保存”或“多条世界线并存”，而不是每次回退都截断未来。
