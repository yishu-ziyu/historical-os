# Round 021 - 启用 Superpower-style 交互规划板

日期：2026-05-18

## 本轮问题
用户希望启用 Superpower 来做交互规划，并在 cmux 侧边栏打开 HTML 作为指导。

## 当前环境发现
- cmux 可用：`cmux 0.62.2`。
- 当前 Claude Code 可用 skills 中没有直接名为 `Superpower` 的 skill。
- 本机存在 OpenCode Superpowers 插件缓存，但不是当前 Claude Code 的直接 skill 入口。

## 本轮执行
创建了一个 Superpower-style 交互规划 HTML：

```text
artifacts/interaction_planning_board.html
```

本地服务：

```text
http://127.0.0.1:8891/interaction_planning_board.html
```

已用 cmux 打开侧边栏：

```text
cmux browser open-split http://127.0.0.1:8891/interaction_planning_board.html
```

## 新协作方式
后续讨论不再先堆抽象术语，而采用“看一幕、改一幕”：

1. 玩家第一眼看到什么？
2. 玩家自然想点哪里？
3. 玩家感觉如何：紧张、好奇、困惑、像工作、想退出？

## 规划板用途
- 显示当前第一案共识。
- 展示玩家第一分钟看到的文案。
- 对比“值班台”与“案情板”等不同呈现方式。
- 作为 cmux 侧边栏里的共同视觉参照。

## 下一步
基于规划板继续做第 22 轮：比较“值班台式”和“案情板式”哪一个更适合作为第一分钟画面。
