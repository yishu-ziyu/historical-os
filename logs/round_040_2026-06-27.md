# Round 040 - Case State 引擎 + 多回合压力测试

> 日期：2026-06-27
> 主题：实现累积 case state 引擎 + 5 回合压力测试

## 实现内容

### Case State 引擎（server-side 持久化）

**新增 server.mjs 模块：**
- `caseStates: Map<caseId, CaseState>` — server 端 case 状态存储
- `createInitialCaseState()` — 初始化
- `getOrCreateCaseState()` — 获取或创建
- `applyStateChange()` — 把单条 stateChange 应用到 case state
- `applyAftermath()` — 批量应用（newNodes/newEdges/newClues/stateChanges/historyFlags）
- `compileWorldStateFromCase()` — 编译为前端消费的 worldState
- `describeStatus()` — 状态枚举值中文化

**初始 case state 包含异常锚点节点**（确保 fallback 路径下也有至少 1 个节点）

### 新增端点

- `GET /api/case/:caseId` — 查询累积 case state
- `POST /api/case/reset` — 重置 case state

### `runTurn` 改动

- 启动时从 caseStates 读累积 state，编译为 worldState
- 结束时把 situation room 的 nodes/edges/riskComponents/historyFlags 合并回 case state
- 返回 `worldState` 字段（前端可直接用）

### `runTurnAftermath` 改动

- 接收 `caseId` 参数
- 调 `applyAftermath()` 把 LLM 返回的 stateChanges 应用到 case state
- 返回 `appliedStateChanges` 和最新的 `worldState`

## 多回合压力测试

### 单元测试（5/5 全绿）

| 测试 | 验证内容 |
|------|---------|
| `case state accumulates nodes and edges across turns` | 2 回合后节点累积 |
| `case state tracks risk components and overall` | 风险值追踪 |

| `5 consecutive turns maintain consistent state` | 5 回合稳定性 |
| `worldState is returned on every turn` | 每回合都返回 worldState |

### 真实模型多回合验证（MiniMax-M3）

**T1 + Aftermath 1 结果：**
- nodes 从 1（锚点）累积到 **6 个**：
  - 锚点 + 普朗克学术庇护斡旋 + 盖世太保非正式监视 + 柏林大学物理讲席秩序 + character.einstein
- edges: 3 个
- stateChangeHistory 累积 **3 条历史合理性变更**：
  - `character.einstein.movement: publicly_active → withdrawing_from_public`
  - `character.planck.role: neutral_colleague → active_intermediary`
  - `environment.berlin_university_physics.safety_level: tenuous_protection → deteriorating`

**关键发现：** 状态变更不是抽象的字段名，而是有历史叙述的语义（"从公共活跃到隐退"、"从中立到积极斡旋"），符合 Story2Game 论文的"前置条件和效果"设计。
