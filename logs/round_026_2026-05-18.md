# Round 026 - 故事网升级为多世界线并存

日期：2026-05-18

## 用户反馈
用户明确选择故事网回退逻辑：

```text
点旧节点
→ 回到旧节点
→ 旧未来不删除
→ 新分支并存
```

## 本轮修复
将 Web Story Loop Demo 的故事历史从线性路径升级为树状/网状结构。

修改文件：

```text
artifacts/web_story_loop_demo/script.js
artifacts/web_story_loop_demo/style.css
```

## 新行为
1. 每个故事节点都有独立 `id`、`parentId`、`childIds`。
2. 点击故事网里的旧节点，只是把它设为当前节点。
3. 旧节点之后已经生成的未来不会被删除。
4. 从旧节点继续选择，会新增一个并存分支。
5. “回到上一步”现在回到当前节点的父节点，而不是删除历史。
6. 故事网用缩进显示父子层级。

## 产品含义
这个版本更接近用户说的“网状历史”：

- 历史不是一条线。
- 回看旧节点不会抹掉已经生长出的未来。
- 玩家可以从同一个历史节点长出多个版本。
- 同一个起点可以被反复重写和探索。
