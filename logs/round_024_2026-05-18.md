# Round 024 - 下一步选择：先做 Web Demo

日期：2026-05-18

## 本轮问题
LLM 驱动的网状历史故事书第一版 Demo 应该先做网页还是 Godot？

## 用户回答
用户选择先做网页。

原因：
- 分享性更强。
- 更适合快速验证 LLM 网状故事循环是否好玩。

## 本轮确认判断
下一步从 Godot 情报工作台原型，转向 Web Demo：

> LLM Story Loop Web Demo v0.1

## Demo 目标
验证最小可玩循环：

```text
历史异常事件
→ 玩家选择预设分支或自由输入
→ 系统生成后续故事
→ 系统给出新的故事线
→ 玩家继续选择
→ 形成网状历史
→ 玩家可以回退
```

## 第一案
事件：

> 1933 年，爱因斯坦没有离开德国。

初始选项示例：
- 爱因斯坦自杀了。
- 爱因斯坦被纳粹杀害。
- 爱因斯坦被迫加入德国研究机构。
- 爱因斯坦通过秘密渠道逃往其他国家。
- 用户自由输入。

## 核心评价标准
- 好玩。
- 好看。
- 能快速分享。
- 能感受到故事不是固定树，而是在历史框架里自由生长。

## 下一步
先进行 BDD 行为对齐，再实现 Web Demo。
