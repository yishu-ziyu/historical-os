# Round 018 - 第一屏范式：情报工作台为主，保留 OS 元素

日期：2026-05-18

## 本轮问题
第一屏信息呈现应采用哪种视觉 / 交互范式？

候选：
1. 危机指挥室式
2. 桌面 OS 式
3. 情报工作台式
4. 档案审计式

## 用户回答
用户确认：可以先做情报工作台式，但也要多一些 OS 式的元素。

## 本轮确认判断
第一案第一屏采用：

> 情报工作台式为主，OS 式元素为辅。

## 产品含义
这不是完整桌面启动器，也不是纯粹单案调查板，而是一个“历史异常值班工作台”。

它应同时具备：

### 情报工作台特征
- 中心有当前案情 / 情报卡。
- 周围有 Timeline、Map、Archives、Risk 等关联面板。
- 玩家围绕一个异常事件做判断。
- 视觉上有任务压力、来源可信度、风险标签、待核验材料。

### OS 式元素
- 顶部状态栏 / 系统时间 / 世界线状态。
- 多窗口或可展开面板。
- 未读通知。
- 文件夹 / 档案夹隐喻。
- 系统审计提醒。
- 弱推荐像系统建议，不像游戏教程。
- 后续可以打开更多 App 或模块。

## 第一屏结构草案

```text
┌──────────────────────────────────────────────┐
│ Top Bar: Historical OS / 1933-03-XX / Baseline Drift │
├───────────────┬────────────────────┬─────────┤
│ Timeline      │ Intel Desk          │ Map     │
│ 异常节点       │ 主情报卡              │ 柏林等地点 │
├───────────────┴────────────────────┴─────────┤
│ Archives: 待核验证据 / Source Credibility / Risk │
└──────────────────────────────────────────────┘
```

更准确地说：
- Intel Desk 是视觉焦点。
- Timeline 和 Map 是被情报点亮的上下文面板。
- Archives 是证据入口。
- Risk / Source Credibility 是决策压力来源。
- OS 元素负责让它像一个可长期运行的系统，而不是一次性任务界面。

## 与前几轮的关系
这个选择平衡了两种需求：

1. 不要一开始就做完整桌面 OS，避免混乱。
2. 不要过度收窄成单案调查游戏，保留 OS 的长期扩展感。
