## 已完成的关键进展

### 1. 产品方向

- 产品不是传统固定剧情 Galgame，也不是普通像素 RPG。
- 核心方向是“会自动演化的架空历史模拟 OS / 网状历史故事书”。
- 玩家可以通过档案、地图、报纸、电报、年鉴、人物关系、行动地点等媒介理解并介入历史。
- 当前第一案仍是：`1933 年，爱因斯坦没有离开德国`。

### 2. 视觉与玩法纠偏

- 已确认主视觉方向：档案素描 / 情报档案 OS。
- 纸上兵棋地图和极简像素地图只作为局部组件语言，不作为全屏主风格。
- 后来又验证了“可行走的历史选择空间”：玩家操控爱因斯坦小人走到地点，按空格触发历史分叉。
- 这个走路层不是完整 RPG，而是空间化选择入口；分叉后仍要回到档案、报纸、电报、地图反馈和历史审计。

### 5. Agent Runtime v0.3 规格

已冻结 v0.3 架构方向，见：

```text
tasks/historical_runtime_v0.3_agent_runtime_spec.md
logs/round_032_2026-05-20.md
```

关键决策：

- Agent 不是故事角色，也不是聊天窗口。
- 最小单位是可审计的 `AgentRun`。
- `HistoricalRuntime` 是事件驱动的 Agent 编排器。
- v0.3 先用固定串行任务图：

```text
StoryWeaverAgent -> HistoryGuardAgent -> IntelDeskAgent -> ArtifactAgent -> Runtime commit
```

- 新 API 方向：

```text

```
