# Round 030 - 行走式历史分叉原型

## 触发

用户看到 Godot 农场/顶视角教程后提出新想法：能否把架空历史游戏也做成可操控人物行走的形式，让爱因斯坦走到不同地点触发不同历史事件，例如离开德国、留在德国、被纳粹抓捕风险等。

## 外部参考判断

- 农场教程的关键价值不是农场题材，而是顶视角移动、可交互地点、触发操作、状态反馈这一套机制。
- Sprout Lands 免费素材包不应直接作为项目默认资产使用；其免费包适合学习/非商业试验，正式项目需要确认授权或重新设计历史题材资产。
- Godot 对应实现路径清晰：`CharacterBody2D` 负责玩家移动，`Area2D` 负责地点触发，`Control` 负责档案/报纸/情报 UI，分叉数据应放在 JSON 或 Resource 中。

## 玩法闭环

```text
移动角色
  -> 进入地点热区
  -> 空格触发分叉
  -> 右侧档案卡更新
  -> 生成下一步 UI 形态提示
```

当前地点：

- 汉堡港撤离线：安排离开德国。
- 普鲁士科学院：留下德国继续发声。
- 秘密警察档案室：触发抓捕风险线。
- 柏林报社：制造舆论保护。

## 判断

这个方向可行，但边界必须清楚：

- 它不是完整像素 RPG。
- 它是“空间化历史选择入口”。
- 走路层负责让历史分叉变得可感、可探索。
- 真正的后续仍应回到档案、报纸、电报、地图和历史审计。

## 美术接入

本轮使用 imagegen 生成第一版资产表，并复制到项目资产目录。原始生成文件保留在 Codex generated_images 目录，项目内使用副本。

当前实现是概念表切片复用：

- 背景叠加柏林视觉。
- 小人优先从资产表取帧，失败时回退到代码绘制小人。
- 地点热区增加交互标记。
- 转场时叠加结果插图区域。

后续如果继续，应生成更规范的独立资产，而不是长期依赖概念表切片：

- 单独的 Einstein top-down sprite sheet。
- 单独的 Berlin street tile/background。
- 单独的 4 个地点图标。
- 单独的 4 张分叉结果插图。

## 抖动修复

用户指出爱因斯坦小人站立时一直左摇右晃。根因是资产接入时每帧都循环切换生成概念表中的不同小人帧，同时又叠加了上下浮动。生成图里的帧没有严格像素对齐，站立时循环播放会读成抖动。

修复：

- 移除持续上下浮动。
- 新增 `player.moving` 状态。
- 静止时固定使用第 1 帧。
- 只有移动时才播放轻微 2 帧走路动画。

验证：

- `node --check artifacts/walking_branch_demo/script.js` 通过。
- 浏览器重新加载后，对玩家附近区域间隔 900ms 做 canvas 像素比较，静止状态 `changedPixels: 0`。
- 新截图：`artifacts/walking-branch-demo-art-v1-still-fixed.png`。

## 角色消失修复

用户指出修复后爱因斯坦本人不见了，画面里出现了资产表白底文字块。根因是从 imagegen 概念资产表中硬切 sprite 的坐标不可靠，实际裁到了说明文字和角色边缘。

修复：

- 不再从概念资产表裁剪玩家角色。
- 玩家角色改为稳定的代码绘制“爱因斯坦小人”：白发、深色外套、脸部和腿部细节。
- imagegen 资产表仅继续用于背景氛围和转场插图，不承担玩家角色 sprite。

验证：

- `node --check artifacts/walking_branch_demo/script.js` 通过。
- 浏览器玩家区域检查白底像素为 0，肤色/深色角色像素存在。
- Chrome 控制台无 error/warn。
- 新截图：`artifacts/walking-branch-demo-player-fixed.png`。
