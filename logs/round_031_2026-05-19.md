# Round 031 - Godot 正式项目壳

## 触发

用户确认下一步进入正式 Godot 开发，不再只在网页原型中试。

## 外部资料依据

- Godot 官方文档确认 2D 玩家移动适合用 `CharacterBody2D`。
- Godot 官方文档确认触发区域可用 `Area2D` 检测 body enter/exit。
- Godot 官方文档确认 UI 层可用 `Control` 节点体系；本项目用 `CanvasLayer` 承载档案 UI。
- AI 辅助 Godot 开发的合理方式是围绕小场景、脚本、数据和可验证运行结果迭代，不直接修改 Godot 引擎源码。

## 产物

```text
artifacts/godot_walking_branch/
  project.godot
  README.md
  scenes/Main.tscn
  scripts/Main.gd
  data/branches.json
```

## 实现

- 新建独立 Godot 4.6 项目，不修改 `游戏引擎/godot` 源码。
- 使用 Compatibility renderer。
- `CharacterBody2D` 玩家，方向键移动。
- 4 个 `Area2D` 分叉地点：港口撤离线、普鲁士科学院、秘密警察档案室、柏林报社。
- 空格触发当前地点。
- 右侧 `CanvasLayer` 档案 UI 更新分叉标题、结果、稳定度、风险和下一步形态。
- 分叉文本和位置从 `data/branches.json` 读取。

## 验证

```bash
godot --headless --path artifacts/godot_walking_branch --check-only --script res://scripts/Main.gd
godot --headless --path artifacts/godot_walking_branch --quit-after 2
```

结果：

- 脚本检查通过。
- 主场景短启动通过。
- 第一次运行因沙盒阻止 Godot 写 `~/Library/Application Support/Godot` 崩溃，放行 `godot --headless` 后正常。
- `.ttc` 字体未导入时不能直接 `load()`，已移除字体硬依赖，先用 Godot 默认字体/fallback。
