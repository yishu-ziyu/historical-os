# Round 037 - 三段式回合循环后端实现

> 日期：2026-06-27
> 主题：实现情报卡 → 沙盘推演 → LLM 叙事推演的三段式回合循环

## 实现内容

### 新增文件
- `data/intel-card.schema.json` — 情报卡数据结构（档案体）
- `data/situation-room.schema.json` — 态势面板数据结构
- `data/action-options.schema.json` — 行动选项数据结构
- `data/state-changes.schema.json` — 状态变更声明数据结构
- `data/spec.md` — 四个 schema 的关系和数据流转说明
- `turn_cycle.test.mjs` — 4 个新测试

### 修改文件
- `server.mjs` — 新增三段式 prompt + 解析器 + 编排器 + 端点

### server.mjs 新增内容

**三个 Prompt Builder：**
1. `buildBriefingPrompt` — 生成情报卡（档案体：编号、日期、来源、正文、手写注释、矛盾点）
2. `buildSituationRoomPrompt` — 生成态势面板 + 行动选项（基于当前 world state）
3. `buildAftermathPrompt` — 生成叙事推演 + 状态变更声明

**三个解析器：**
1. `parseBriefingResult` — 解析 LLM 输出为 intel-card 结构
2. `parseSituationRoomResult` — 解析 LLM 输出为 situation-room + action-options 结构
3. `parseAftermathResult` — 解析 LLM 输出为 narrative + state-changes 结构

**两个编排器：**
1. `runTurn` — 串联 Stage 1 (Briefing) + Stage 2 (Situation Room)，带 fallback
2. `runTurnAftermath` — Stage 3 (Aftermath)，带 fallback

**两个新端点：**
1. `POST /api/turn/start` — 启动一回合，返回 briefing + situationRoom
2. `POST /api/turn/aftermath` — 执行 aftermath，返回 narrative + stateChanges

**保留向后兼容：**
- `POST /api/generate` — 旧端点，直接返回 story + choices
- `POST /api/generate/start` — 旧异步端点，走 legacy 流程

### 测试结果
- 4 个新 turn_cycle 测试：全绿
- 5 个原有非 timeout 测试：全绿
- 1 个 timeout 测试：预存失败（与本次改动无关）
