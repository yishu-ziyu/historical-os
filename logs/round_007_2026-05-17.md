# Round 007 - 历史异常事件模板优先级

日期：2026-05-17

## 本轮问题
什么样的“历史异常事件”最适合作为 OS 默认入口？

候选类型：
- 电报异常
- 人物异常
- 战役异常
- 档案异常
- 地图异常
- 舆论异常
- 技术异常

## 用户回答
用户基本同意优先选择“档案异常 + 电报异常”的判断，并补充：

- 人物异常也可以，也比较适合。
- 技术异常会非常宏大，说不定有些用户会喜欢这一版。

## 本轮确认判断
MVP 第一批异常事件模板优先级：

### 第一优先级
1. 档案异常
   - 一份档案与真实历史不一致。
   - 适合历史调查者入口。
   - 强证据感，最能体现 Archives 的价值。

2. 电报异常
   - 一封不该存在、提前出现、延迟送达或被篡改的电报。
   - 适合外交官 / 信使 / 情报人员入口。
   - 天然激活 Intel Desk。

3. 人物异常
   - 关键人物提前死亡、失踪、倒戈，或活过原本死亡时间。
   - 适合调查、情报和政治惊悚。
   - 容易产生情感与叙事牵引。

### 扩展优先级
4. 技术异常
   - 某项技术提前出现或被延迟。
   - 更宏大，适合世界线长期演化。
   - 可能吸引喜欢宏观历史模拟的用户。

### 暂不作为 MVP 核心
- 战役异常：戏剧性强，但容易把产品带向军事推演。
- 地图异常：适合作为异常表现或线索，但单独作为入口可能需要更多解释。
- 舆论异常：很有价值，但伦理边界和机制复杂，后续再深挖。

## 产品含义
第一批异常事件模板应同时服务四个核心区：

```text
Timeline：异常与真实历史时间点的差异。
Map：异常涉及的地点、路线、国家或战区。
Archives：原始证据、史料、档案、人物资料。
Intel Desk：待判断情报、行动选项、风险提示。
```

## 下一轮问题
第 8 轮：第一批异常模板里，最适合做“新手第一案”的是哪一个？

候选：
1. 档案异常第一案：档案馆出现一份与真实历史冲突的会议记录。
2. 电报异常第一案：收到一封本不该存在的加密电报。
3. 人物异常第一案：一个关键人物提前死亡/失踪/活过死亡时间。
4. 技术异常第一案：某项技术提前出现，引发世界线偏移。
