# 历史态势决策调研

日期：2026-05-18
来源：历史态势决策调研 Agent

## 研究对象

- Suzerain
- Democracy 4
- Terra Invicta
- Victoria 3
- Crusader Kings 系列

## 1. 玩家为什么喜欢历史态势决策

核心不是“看剧情”，而是“在历史压力下做选择”。

Suzerain 的吸引力在于总统办公室式决策：改革、财政、军方、外交互相牵制。

Democracy 把政策影响拆成选民群体、满意度、指标链。

Terra Invicta / Victoria / Crusader Kings 提供“世界在动”的态势感。

玩家喜欢的是：

- 有限信息
- 不可逆后果
- 变量牵连
- 历史 if 线
- “我改变了大势”的责任感

## 2. 什么会让产品像表格工作

危险点是把变量直接摊开：GDP、军力、民意、派系好感全变成可优化 KPI。

糟糕体验包括：

- 每回合只是点政策、看数值涨跌。
- 事件没有人、地点、媒体反馈。
- 地图只是筛选器。
- 档案只是百科。
- Democracy 式系统如果缺少叙事包装，会像政策 Excel。
- Victoria 式经济链容易让非硬核玩家觉得在调参。

## 3. 对本项目第一屏的建议

第一屏四区合理，但要按“危机指挥室”而非“功能菜单”设计。

- Timeline：不是历史年表，而是倒计时压力线，显示即将恶化的节点。
- Map：显示势力温度、风险扩散、关键人物位置，不做完整大战略地图。
- Archives：只放可用于判断的证据、报纸、信件、人物档案。
- Intel Desk：当前案情摘要、矛盾假说、未确认情报。

## 4. 对五个动作的建议

Agent 建议的动作框架：

1. 阅读档案：获得证据，不直接加数值。
2. 召见人物：改变关系与信息来源。
3. 发布立场：影响舆论、政党、国际观察。
4. 调配资源：保护 / 监控 / 撤离 / 资助某对象。
5. 推进时间：结算连锁后果，生成新态势。

## 5. 对第一案的建议

“1933 爱因斯坦未离德”应避免恋爱线，重点做成多方压力局：

- 科学家
- 纳粹政权
- 国际学界
- 犹太社群
- 核研究早期网络
