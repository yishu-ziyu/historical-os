# 混合式三段展开 — 回合循环设计

> 2026-06-27 起草。基于第 35 轮调研结论（TNO / Kaiserreich / Papers Please / Disco Elysium）。

---

## 一、核心问题：一回合 = 一次决策，三段展开

采用 **一次决策，三段展开**。

理由：

- 参照物 TNO 的核心体验不是"同时处理 10 件事"，而是"收到一个超事件 → 做关键选择 → 看后果"。
- 参照物 Papers Please 的核心节奏是"审一个人 → 盖章 → 看后果 → 下一个"。
- 参照物 Disco Elysium 的核心节奏是"面对一个场景 → 选一条对话/行动 → 看结果 → 下一个场景"。

这三个游戏的共同模式：**一次聚焦的决策，三段有节奏的反馈。**

你的游戏不应该是"指挥官同时管五件事"，而是"情报分析师处理一张异常卡——然后看世界怎么回应"。

---

## 二、一回合的三段

### 第一段：情报卡（The Briefing）— 约 2-3 分钟

**你在看什么：**
一张「情报卡」落到 Intel Desk。它不是 A/B 选项，而是一份需要你阅读、理解、标记的档案。

**情报卡的固定结构（参照 Kaiserreich 档案体）：**

```
┌─────────────────────────────────────┐
│ 【绝密】柏林科学院内部通信          │
│ 编号：B-GW-1933-0472               │
│ 日期：1933 年 10 月 15 日           │
│ 来源：德国物理学会存档              │
│                                     │
│ 正文：                              │
│ 10 月 15 日常务会议记录显示，      │
│ 所有犹太裔会员名单被单独归档至     │
│ "特别登记册"。但 A. Einstein 的    │
│ 名字仍在完整会员名册中。            │
│                                     │
│ 会议纪要手写注释栏：                │
│ "Herr Professor wünscht zu        │
│  bleiben." （教授希望留下。）       │
│                                     │
│ ─── 你的分析标记区 ───              │
│ ○ 关键人物状态异常                  │
│ ○ 时间节点异常（柏林会议尚未召开）  │
│ ○ 信息来源待验证                    │
│                                     │
│ [标记线索] [验证来源] [发送指令]    │
└─────────────────────────────────────┘
```

**你在做什么：**
- 阅读卡面文本（参照 Disco Elysium 的文本阅读 UX）
- 标记你认为是关键线索的片段（高亮、标注）
-  optionally 打开关联档案做交叉比对

**核心设计信号：**
- 文本本身就要有推理价值（Kaiserreich 式埋线索，不直接告诉玩家"这很危险"）
- 操作是"标记"，不是"选 A/B"（参照 Papers Please 的盖章感）
- 最多 3 个操作入口，不堆选项

---

### 第二段：沙盘推演（The Situation Room）— 约 1-2 分钟

**你在看什么：**
你标记完线索后，情报卡"下沉"到 Situation Room 层。这里是一张简化的态势面板——不是大地图，而是关键节点和关系网络。

**参考 TNO 的超事件链结构：**

```
异常锚点 ──→ 爱因斯坦未离开德国
                 │
       ┌─────────┼─────────┐
       ▼         ▼         ▼
   柏林科学院   铀计划团队  国际舆论
       │         │         │
  会员名单异常  技术路线   爱因斯坦公开声明
       │         │         │
       └─────────┼─────────┘
                 ▼
           [你的选择在此]
```

**你在做什么：**
- 选择一个行动方向
- 看系统生成的"后果预览"——不是详细叙事，而是状态标签

**行动选项设计（参照 TNO 封锁式设计）：**

不是"你想做什么都行"，而是 3-4 个有历史框架内合理性的方向：

| 选项 | 历史合理性 | 后果倾向 |
|------|-----------|---------|
| 秘密接触爱因斯坦 | 他本人未离开，有接触窗口 | 高风险、高信息回报 |
| 监控柏林科学院内部通讯 | 德国学术系统有通道 | 低风险、低即时回报 |
| 借舆论压力公开此事 | 国际媒体可施压 | 中等风险、影响不可控 |
| 暂不行动，继续观察 | 等更多信息 | 安全但异常在自行演化 |

**关键：没有"正确答案"。每个选项都导向不同的后续事件链，且后果有延迟。**

**参照 Papers Please 的时限感：** 你的选择不是无限的——报纸出刊、电报到达、会议召开这些叙事性的截止线在逼近。不是倒计时条，是情报卡上已经出现的"下一个时间节点"。

---

### 第三段：LLM 叙事推演（The Aftermath）— 约 3-5 分钟

**你在看什么：**
系统根据你的选择，生成一段叙事文本——但**不是自由续写，而是结构化输出**（参照 Story2Game 论文的设计）。

**输出结构（参照 Hidden Door 的卡牌叙事层）：**

```
┌─────────────────────────────────────┐
│ 情报值班台 · 柏林通讯第 7 号         │
│ ─────────────────────────────────── │
│                                     │
│ 你选择了"秘密接触"。                │
│                                     │
│ 10 月 17 日凌晨，柏林大学校区       │
│ 侧门。一位自称来自"普鲁士文化       │
│ 协会"的联络人进入爱因斯坦办公室。   │
│                                     │
│ 根据你的内部报告，接触持续了        │
│ 47 分钟。联络人提出了两项邀请：      │
│ 一项关于继续在柏林工作的条件，       │
│ 一项关于参与一项"国家科学计划"      │
│ 的邀请。                            │
│                                     │
│ 爱因斯坦没有当场答复。他说需要       │
│ "想想"。                            │
│                                     │
│ ─── 世界线偏移标记 ───              │
│ ▸ 爱因斯坦状态：犹豫中（原历史：    │
│   已开始流亡准备）                   │
│ ▸ 柏林科学院：出现非官方接触渠道    │
│ ▸ 风险指数：上升（+1）              │
│ ▸ 公开风险：低（接触未暴露）        │
│                                     │
│ [情报卡新增：爱因斯坦 10/17 会见   │
│  记录（草稿）]                       │
│ [新分支：48 小时后待触发]           │
│ [继续下一回合]                      │
└─────────────────────────────────────┘
```

**你在做什么：**
- 阅读生成的叙事
- 看世界线偏移标记——你的选择确实改变了什么
- 接收新的情报卡（可能是下一回合的 Briefing）
-  optionally 自由文本输入一个补充指令

**参照 Story2Game 的核心机制：** LLM 的每次输出都附带**可执行的状态声明**。不是"写了一段好看的话"，而是"爱因斯坦状态变成犹豫中，柏林科学院出现新渠道，风险+1"。这些状态变更被引擎执行，影响后续所有情报卡。

**参照 Hidden Door 的约束：** 如果玩家输入"我派特工暗杀爱因斯坦"，AI 会判断这个行动在当前历史框架内是否可行。如果不可行，不是直接拒绝，而是给出**条件反馈**——"这个方向需要额外条件：你需要先获得行动授权，且当前柏林安全环境不支持"。

## 三、完整的回合循环（以爱因斯坦案为例，从头到尾跑一遍）

### 回合 1 — "爱因斯坦为什么还在柏林"

**第一段：情报卡**
```
情报卡落到 Intel Desk：
"柏林物理学会 1933 年 10 月 15 日会议记录...
Herr Professor wünscht zu bleiben."
```
→ 玩家标记关键线索：爱因斯坦名字仍在会员名册 + 手写注释"希望留下"
→ 点击「验证来源」

**第二段：沙盘推演**
```
态势面板出现：
异常锚点：爱因斯坦未离开德国
关联节点：柏林科学院 / 铀计划团队 / 国际舆论

4 个行动选项：
[秘密接触] [监控通讯] [舆论施压] [继续观察]
```
→ 玩家选择「秘密接触」
→ 系统显示后果预览："高风险，可能获得直接对话窗口"

**第三段：LLM 推演**
```
叙事输出：
10/17 凌晨接触场景...
爱因斯坦"需要想想"...

世界线偏移：
▸ 爱因斯坦状态：犹豫中
▸ 柏林科学院：出现非官方接触渠道
▸ 风险指数：+1

新增情报卡：爱因斯坦 10/17 会见记录（草稿）
下一个时间节点：10 月 20 日
```
→ 回合结束。玩家点击「继续下一回合」。

---

### 回合 2 — "他想留下来"

**第一段：情报卡**
```
新卡：「德国文化部内部备忘录」
"关于 A. Einstein 参与国家科学计划
 的可能性评估..."

备忘录提到：爱因斯坦已被列为
"关键技术人员"保护名单。
```
→ 玩家标记：爱因斯坦状态从"犹豫"变成"被保护"
→ 交叉比对第一回合的情报卡，发现矛盾：手写注释说"希望留下"，但备忘录是单方面的"保护"

**第二段：沙盘推演**
```
态势面板更新：
▸ 爱因斯坦状态：被保护（强制）
▸ 柏林科学院：非官方接触已记录
▸ 新的关联节点出现：德国文化部

3 个行动选项（上一回合的选择影响了可选范围）：
[警告爱因斯坦风险] [获取文化部更多情报] [评估铀计划关联]
```
→ 玩家选择「获取文化部更多情报」

**第三段：LLM 推演**
```
叙事输出：
文化部档案库检索结果...
一份 10 月 18 日的内部文件：
"Einstein 的学术自由将被限制，
但不会被强制离开。条件是：
继续领导理论物理研究所。"

世界线偏移：
▸ 爱因斯坦状态：受限但留守
▸ 柏林科学院：被文化部纳入管理
▸ 风险指数：+2（接触已暴露）
▸ 新线索：理论物理研究所 = 铀计划前身？

新增情报卡：文化部 10/18 文件
下一个时间节点：10 月底
```
→ 回合结束。

---

### 回合 3 及以后 — 世界线在生长

从第三回合开始，态势面板的复杂度自然增长：

- 第一回合只有 3 个关联节点
- 第三回合出现 5-6 个节点，部分节点之间有连线
- 玩家的选择开始产生**连锁效应**——"秘密接触"暴露 → 文化部记录 → 铀计划关联 → 国际舆论反应

**参照 TNO 的事件链设计：** 同一异常的不同路径：
```
路径 A：爱因斯坦成功离开 → 曼哈顿计划加速 → 美国先造出原子弹
路径 B：爱因斯坦留下但被监控 → 铀计划德国主导 → 时间线大幅偏移
路径 C：爱因斯坦公开反对纳粹 → 被驱逐但世界看到 → 舆论压力改变外交格局
```

**参照 Papers Please 的多结局：** 每个选择都有后果，但后果不是立即显现的。像 Papers Please 的"家庭场景"——你今天盖的每一个章，几周后才会在家庭场景里显现。

## 四、回合循环的核心规则

| 规则 | 说明 | 参照来源 |
|------|------|---------|
| **每次输出都产生状态变更** | LLM 叙事不是自由文本，每段都附带可执行的状态声明 | Story2Game |
| **情报卡文本本身就是线索** | 不直接告诉玩家"发生了什么"，让玩家从文本拼出来 | Kaiserreich |
| **操作有体感** | 标记/盖章/归档，不是点按钮弹文字 | Papers Please |
| **选项有历史框架约束** | 不是"你想做什么都行"，是"在历史框架内你能做什么" | TNO 封锁式设计 |
| **后果有延迟** | 今天的选择在 2-3 回合后才显现，像盖的章 | Papers Please |
| **叙事文本像真实档案** | 有日期、编号、手写注释、矛盾点 | Kaiserreich 档案体 |
| **文本排版是玩法的一部分** | 行距、段间距、字体选择影响阅读体验 | Disco Elysium |
| **自由文本输入受约束** | 玩家可以写补充指令，但不符合历史框架的会被条件化拒绝 | Hidden Door |

## 五、跟现有代码的衔接

当前 `web_story_loop_demo/` 的架构已经支持这个循环：

- `POST /api/generate/start` → 生成情报卡（Briefing）
- `GET /api/jobs/{jobId}` → 获取状态和事件
- Agent Runtime 的 `StoryWeaverAgent` → 负责叙事生成
- `HistoryGuardAgent` → 负责约束在历史框架内
- 前端的前台/技术事件分层 → 区分玩家可见和后台审计

**需要新增的：**
1. 情报卡的「档案体」文案模板（JSON schema）
2. 态势面板的数据结构（节点 + 连线 + 状态标签）
3. 状态变更声明格式（LLM 输出的结构化部分）
4. 「操作体感」的前端实现（标记高亮、拖拽归档、盖章动效）
